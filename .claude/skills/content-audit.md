# /content-audit — 审计并修复内容质量

扫描所有内容，找出质量问题并自动修复。

## 使用方法
```
/content-audit
/content-audit prompt-engineering
```

## 审计标准

对每个 .md 文件检查以下项目：

| 检查项 | 不合格标准 | 修复方式 |
|--------|-----------|---------|
| 配图 | 0 张图 | WebSearch 找相关图片，补上 |
| 字数 | < 400 字 | 搜索补充内容，扩写到 500+ |
| AI 味词 | 含禁止词 | 替换为口语化表达 |
| 代码示例 | 没有代码块 | 补充可运行的示例 |
| 来源链接 | 文章没有来源 | 搜索补充原文链接 |
| frontmatter | 缺 description/tags | 补上 |

### AI 味禁止词清单

自动搜索并替换：
- "值得注意的是" → 删掉，直接说
- "总的来说/总而言之" → "说白了" 或删掉
- "此外/另外/与此同时" → "还有个事"、"顺便说" 或直接另起段
- "综上所述" → 删掉
- "不可否认" → 删掉
- "至关重要/具有重要意义" → 说具体为什么重要
- "旨在/致力于" → "就是为了"
- "使得/使其" → "让"
- "进行了/进行" → 直接用动词

## 执行步骤

1. 扫描 `src/content/` 下所有 .md 文件
2. 对每个文件跑审计检查，输出报告表格
3. 按严重程度排序（0 图 > 太短 > AI 味词）
4. 逐个修复：
   - 0 图：WebSearch 找相关配图 URL，插入文章合适位置
   - 太短：WebSearch 搜索该主题最新内容，扩写
   - AI 味词：全局搜索替换
   - 缺代码：补充真实可运行的示例
5. 修复完输出对比报告

## 输出格式

```
📊 内容质量审计报告

| 文件 | 图片 | 字数 | AI词 | 代码 | 状态 |
|------|------|------|------|------|------|
| cursor-tips.md | 0→2 | 380→650 | 0 | ✅ | 已修复 |
| prompt-eng/01.md | 0→1 | 520 | 0 | ✅ | 已修复 |
| ... |

修复了 {N} 个文件，补了 {N} 张图，扩写了 {N} 篇。
```

## 规则

- 补图用绝对 URL（产品官网、Unsplash）
- 扩写要有真实内容（搜索后写），不要水
- 替换 AI 味词后读一遍确认通顺
- 不改动已经质量合格的文件
