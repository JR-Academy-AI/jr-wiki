## 注册 Manus

Manus 目前有免费方案（每日 300 credits 刷新），不再需要排队邀请码。打开 [manus.im](https://manus.im) 直接注册：

![Manus 仪表盘概览](https://img.youtube.com/vi/wN5lK4zvhns/maxresdefault.jpg)

1. 访问 manus.im → 点击 "Get Started"
2. 用 Google 账号或邮箱注册
3. 进入 Dashboard，你已经有 300 免费 credits 可用

```bash
# 注册前确认你的网络环境
# Manus 服务器在海外，国内直连可能慢
# 建议：稳定的网络环境 + Chrome 浏览器
```

## 界面布局

Manus 的工作界面分三个区域：

```
┌─────────────────────────────────────────────────────┐
│  左侧：对话面板          │  右侧：Agent 工作区        │
│  ┌────────────────┐     │  ┌──────────────────────┐ │
│  │ 你的输入 / 历史 │     │  │ Cloud Browser        │ │
│  │                │     │  │ (实时显示浏览器操作)   │ │
│  │ Agent 状态更新  │     │  │                      │ │
│  │                │     │  ├──────────────────────┤ │
│  │ 交付结果/文件   │     │  │ Terminal / 文件系统   │ │
│  └────────────────┘     │  └──────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

- **左侧对话面板**：输入任务、看 Agent 的思考过程和状态更新
- **右侧 Cloud Browser**：实时显示 Agent 在浏览器里的操作——打开了什么网页、点击了什么按钮
- **Terminal / 文件**：切换查看 Agent 跑的命令和生成的文件

## 两种交互模式

Manus 有两种模式，别选错：

| 模式 | 适合场景 | Credits 消耗 |
|------|---------|-------------|
| Chat Mode | 快速问答、简单翻译、闲聊 | 极低（~2-5 credits） |
| Agent Mode | 需要搜索、写代码、操作文件的复杂任务 | 较高（50-500+ credits） |

默认是 Agent Mode。如果你只是想聊两句，切到 Chat Mode 省 credits。

## 第一个任务：竞品分析

来试一个典型任务——让 Manus 做一份竞品分析报告：

```
帮我分析 3 个澳洲在线教育平台（JR Academy, 匠人学院竞品）：
1. 搜索 Codecademy、Udemy、Coursera 的澳洲业务
2. 对比课程类型、价格、目标用户
3. 生成一个 Markdown 格式的对比表格
4. 给出 3 条差异化建议
```

发送后你会看到：

1. **规划阶段**：Manus 列出执行步骤（"我会先搜索每个平台的信息..."）
2. **执行阶段**：Cloud Browser 开始打开网页，你能实时看到它在浏览哪些页面
3. **整理阶段**：回到终端处理数据、生成文件
4. **交付阶段**：左侧对话面板出现最终报告 + 可下载文件

整个过程 3-5 分钟，消耗约 80-150 credits。

## 理解 Credits 系统

Manus 的计费单位是 credits，不同操作消耗不同：

```python
# Credits 消耗参考（近似值）
credit_cost = {
    "简单问答 (Chat Mode)":     "2-5 credits",
    "单次网页搜索":              "10-15 credits",
    "内容生成 + 调研":           "50-100 credits",
    "全栈应用构建":              "200-500 credits",
    "大规模数据分析":            "500-900 credits",
}

# Free 方案：每天刷新 300 credits
# 够用来做 2-3 个中等复杂度任务
# 或 1 个重度任务（如完整 App 开发）
```

省 credits 的关键：**把需求说清楚**。模糊的指令会让 Agent 多次试错，白白烧 credits。

## 任务结果怎么拿

任务完成后，Manus 会在对话面板里给你结果。如果有文件产出（PDF、代码、Excel 等）：

1. 直接在对话里点击下载
2. 或者在文件面板查看所有产出物
3. 代码类产出可以一键部署（Manus 支持直接部署到公网）

一个 pro tip：如果任务做到一半方向不对，随时在对话框里打断并重新指示。Manus 会根据新指令调整，不用重新开始。

## 常见新手坑

| 问题 | 解决 |
|------|------|
| Agent 卡在加载页面 | 遇到 CAPTCHA 或登录墙了，换个数据源或手动提供信息 |
| Credits 一下子烧完 | 任务描述太模糊，Agent 在试错。写清楚每一步预期 |
| 结果不够深入 | 要求"至少分析 5 个来源"或"每个要点附带数据佐证" |
| 服务器繁忙 | 高峰时段常见，换个时间或升级付费方案 |
