Replit Agent 的上手过程比大多数开发工具简单得多——打开浏览器，注册，输入一句话描述你想要的应用，等 Agent 干完活，点部署。下面一步步走。

![Replit Agent 新手教程](https://img.youtube.com/vi/qpIDMZZIv3s/maxresdefault.jpg)

## 第一步：注册账号

打开 [replit.com](https://replit.com)，点右上角 Sign Up。支持 Google、GitHub、Apple 账号直接登录，或者用邮箱注册。

注册完默认是 **Starter（免费）** 计划：
- 大约 10 次免费 Agent checkpoint（够你完整体验一个小项目）
- 每天有限的 AI 额度
- 最多 10 个开发项目，1 个发布项目
- 代码公开可见（免费用户没有私有项目）

如果你是学生，用 `.edu` 邮箱可以申请学生折扣：Core 计划 $10/月（原价 $25），优惠 6 个月。

## 第二步：创建项目，启动 Agent

登录后你会看到 Replit 的工作台。创建新项目的方式：

1. 点击左上角 **"Create Repl"** 按钮
2. 选择 **"Agent"** 模式（不是 Assistant）
3. 在输入框里用自然语言描述你想做什么

Agent 和 Assistant 的区别：Agent 是自主执行（你说它做），Assistant 是辅助模式（你写它帮）。新手建议直接用 Agent。

## 第三步：写好你的第一个 Prompt

Prompt 的质量直接决定 Agent 的产出质量。下面是一个好 prompt 的例子：

```
请帮我创建一个个人任务管理应用，要求：
1. 用 React + Node.js + PostgreSQL
2. 功能：添加任务、标记完成、删除任务、按日期筛选
3. UI 风格：简洁现代，用 Tailwind CSS
4. 需要用户登录功能（用 Replit Auth）
5. 数据存到 PostgreSQL 数据库
```

**Prompt 写作技巧**：

| 做法 | 好的写法 | 差的写法 |
|------|---------|---------|
| 指定技术栈 | "用 React + Express + PostgreSQL" | "帮我做个 app" |
| 拆分功能 | "1. 登录 2. 增删改查 3. 筛选" | "做一个全功能的任务管理" |
| 定义 UI 风格 | "简洁现代，深色主题" | "好看一点" |
| 给约束条件 | "最多 5 个页面，手机端优先" | （不给任何限制） |

## 第四步：看 Agent 干活

点发送后，Agent 开始工作。你会看到：

1. **规划阶段**：Agent 先列出要做的事（安装依赖、创建文件结构、写代码）
2. **编码阶段**：自动创建文件、写代码、安装 npm 包
3. **测试阶段**：Agent 会自己运行应用、点按钮、测接口
4. **修复阶段**：如果发现 bug，自动修复再测试

整个过程中你可以在右侧实时预览应用的效果。如果 Agent 做的方向不对，随时打断它（点暂停按钮），给新的指示。

每次 Agent 完成一个关键节点，系统会自动创建一个 **Checkpoint**（存档点）。如果后续改坏了，可以一键回滚到之前的状态。

## 第五步：部署上线

应用做好后，点顶部的 **"Deploy"** 按钮：

```bash
# Replit 提供三种部署方式：

# 1. Autoscale（推荐）—— 有流量才收费，没人访问时自动缩到零
#    适合：大部分 Web 应用、API
#    起步价：~$1/月

# 2. Reserved VM —— 24/7 常驻服务器
#    适合：需要稳定在线的应用
#    起步价：~$20/月

# 3. Static —— 纯静态站点
#    适合：个人主页、博客
#    费用：极低
```

部署完成后你会得到一个 `你的项目名.replit.app` 的域名，也支持绑定自定义域名。

## 新手避坑

- **不要一次描述太多功能**：先做核心功能，跑通后再加。Agent 一次接收太多需求容易混乱。
- **看到 Agent 转圈超过 5 分钟**：果断暂停，换个更具体的说法重新来。
- **免费额度用完了别慌**：每日额度第二天重置。想认真用建议升 Core。
- **代码下载到本地**：免费用户的代码是公开的，重要项目建议下载备份或升级到付费计划。
