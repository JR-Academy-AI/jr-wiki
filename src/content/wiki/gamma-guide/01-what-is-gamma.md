Gamma（gamma.app）是一个 AI 驱动的演示文稿 + 文档 + 网站生成平台——你输入一句话描述主题，它在 60 秒内帮你产出一整套带排版、配图和设计的内容，不用碰 PowerPoint，不用学 Figma。

![Gamma AI 编辑器界面](https://images.gamma.app/b4ufuay1lzgjxjt/original/7085c28fa3dd407e9a4359dce67741a0/gamma-header.png)

## 为什么 Gamma 突然火了

三个创始人（Grant Lee、Jon Noronha、James Fox）之前在 Optimizely 做 A/B 测试产品，2020 年出来创业做 Gamma。2025 年 11 月拿了 Andreessen Horowitz 领投的 6800 万美元 B 轮，估值 21 亿美元。到 2026 年初，用户突破 7000 万，年化收入过亿美元——而且是盈利的（2023 年就盈利了，AI 公司里很少见）。

火的核心原因：**把 AI 做成了默认工作方式，而不是附加功能**。你不是先选模板再慢慢填内容，而是直接说"帮我做一个给投资人看的 SaaS 产品 pitch deck"，Gamma 从大纲到配图一步到位。

## 卡片式排版：Gamma 和传统 PPT 的根本区别

PowerPoint / Keynote 是一页一页的幻灯片，16:9 画布，改个字号都要调半天对齐。Gamma 用的是**卡片（Card）** 模式——更接近网页的滚动体验：

```
传统 PPT:
┌─────────┐  ┌─────────┐  ┌─────────┐
│ Slide 1 │→ │ Slide 2 │→ │ Slide 3 │  固定 16:9，翻页
└─────────┘  └─────────┘  └─────────┘

Gamma 卡片:
┌─────────────────────────────┐
│ Card 1（可长可短，自适应内容）  │
├─────────────────────────────┤
│ Card 2（嵌套子卡片、折叠区域）  │
├─────────────────────────────┤
│ Card 3（内嵌视频、图表、Figma）│
└─────────────────────────────┘
  ↕ 上下滚动，像读网页
```

好处很直接：内容多就让卡片长一点，不用为了塞进一页把字缩到看不清。而且卡片里能放折叠区域（Toggle）、嵌套子卡片（Nested Card）这些交互元素，传统 PPT 做不到。

## 和竞品怎么选

| 维度 | Gamma | Canva AI | Beautiful.ai | PowerPoint Copilot |
|------|-------|---------|-------------|-------------------|
| **定位** | AI 优先的内容生成 | 模板优先的设计平台 | 自动排版的 PPT 工具 | 传统 PPT + AI 助手 |
| **生成速度** | 60 秒出完整演示 | 需要选模板再调 | 中等 | 慢 |
| **排版方式** | 卡片式滚动 | 传统画布 | 传统幻灯片 | 传统幻灯片 |
| **免费额度** | 400 AI 积分（约 10 份） | 有限免费功能 | 无免费版 | 需 365 订阅 |
| **导出格式** | PDF / PPTX / 网页链接 | 多格式 | PDF / PPTX | PPTX |
| **开发者 API** | 有（2026.01 正式版） | 有 | 无 | 无 |
| **中文支持** | 支持生成 + 宋体/雅黑优化 | 支持 | 弱 | 支持 |

**我的建议**：需要快速出内容、不想折腾排版 → Gamma；需要精细设计控制 → Canva；已经绑定 Microsoft 生态 → PowerPoint Copilot。

## 开发者为什么该关注 Gamma

你可能觉得"做 PPT"跟开发者没关系。但想想这些场景：

- Sprint demo 演示给 PM 和客户看
- 技术方案评审的架构图文档
- 面试时展示个人项目作品集
- 用 API 批量生成客户报告（Gamma 有完整的 REST API）
- 通过 MCP 协议让 Claude / ChatGPT 直接调 Gamma 生成内容

Gamma 在 2026 年 1 月开放了正式 API，还上线了 MCP Server。这意味着你可以写代码让 AI 自动产出演示文稿——这不是一个只给市场部用的工具。

## 谁适合用 Gamma

**很适合**：
- 经常做 demo / pitch / 汇报的开发者和 PM
- 想快速把想法可视化、不想学设计工具的人
- 需要用 API 批量生成内容的技术团队
- 个人博客或作品集想用网页式呈现的人

**不太适合**：
- 对排版有像素级强迫症的设计师（Gamma 的自由度不如 Figma）
- 需要导出完美 PPTX 的场景（导出会丢交互元素和部分排版）
- 只做内部文档、纯文字不需要视觉的情况
