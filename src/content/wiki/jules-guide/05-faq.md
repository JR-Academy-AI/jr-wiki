## 常见问题

### Jules 任务失败了怎么办？

Jules 的任务成功率大概在 75% 左右。失败通常有几个原因：

![Jules 定价与常见问题](https://img.youtube.com/vi/mLbI96CmV0E/maxresdefault.jpg)

```bash
# 排查清单：
# 1. Prompt 太模糊？→ 加上具体文件路径、函数名、期望结果
# 2. 环境没配好？→ 检查 Environment Snapshot 是否正确
# 3. 项目太大？→ 超过 768K token 的文件会被跳过
# 4. 依赖安装失败？→ 更新 Setup Script 确保安装成功
# 5. "System is experiencing heavy traffic"？→ 高峰期拥堵，等几小时再试
```

**坑**：失败的任务也扣每日配额。免费用户别拿不靠谱的 Prompt 试水，先想清楚再提交。

### 能不能在任务执行中途改方向？

不能。这是 Jules 和 Claude Code 最大的区别——Claude Code 是实时对话，你随时可以说"停，换个方向"。Jules 的 Plan-then-Execute 模型决定了：计划阶段可以反复修改，但一旦 Approve 就是单程票。

所以**计划审查是你最重要的操作**。多花 30 秒看计划，比任务跑完发现不对再重来省时间多了。

### Jules 只支持 GitHub 吗？

对，目前只能连 GitHub。GitLab、Bitbucket、本地 Git 都不行。而且只支持个人 Google 账号（@gmail.com），Google Workspace 企业账号还在等待支持。

### 代码安全有保障吗？

Google 官方承诺：
- **不用你的代码训练模型**
- 代码在隔离的 Google Cloud VM 中处理
- 每次任务用全新的短生命周期虚拟机
- 任务结束后 VM 销毁

但还是那句话：**不要在仓库里提交密钥和 token**。用环境变量，不要硬编码。

### Jules 支持中文交互吗？

官方文档说目前只支持**英语**作为交互语言。实际用中文写 Prompt 也能跑，但准确率不如英文。建议：任务描述用英文，代码注释和文档用中文没问题。

| **AI Pro** | $19.99 | 100 | 15 | 正经干活的起步线，Suggested Tasks 开启 |

**免费用户省额度的技巧**：

1. 把 Prompt 写到 90% 确定了再提交——失败也扣次数
2. 优先提交确定性高的任务（测试生成、代码风格统一、文档补充）
3. 架构级的探索去 Claude Code 或 Cursor 做完决策后，把明确的执行任务交给 Jules
4. 一天 15 次，可以先在早上集中提交 3 个并发，审完 PR 后下午再提交

## 选型决策树

不知道该选哪个？按这个逻辑走：

```
你需要 AI 帮你写代码
├── 需要实时交互、边写边聊？
│   ├── 想在 IDE 里操作？→ Cursor
│   └── 想在终端操作、需要强推理？→ Claude Code
└── 不需要实时，想把任务丢出去收 PR？
    ├── 预算充足、需要浏览器和完整 IDE？→ Devin
    └── 预算有限、能拆成明确小任务？→ Jules ✅
```

最佳组合推荐：**Claude Code 做架构设计和复杂推理，Jules 做批量执行和日常维护。** 两者互补而不是竞争——Claude Code 帮你想清楚，Jules 帮你批量执行。

## Jules V2 前瞻（Project Jitro）

Google I/O 2026（5 月 19 日）预计发布 Jules V2，代号 **Jitro**。从"任务驱动"升级为"目标驱动"：

- 不再是一个个写 Prompt，而是设定 KPI（比如"把测试覆盖率提到 80%"）
- Jules 自己分析代码库、找到差距、逐步推 PR 达成目标
- 跨会话记忆——不会每次都从零开始理解项目
- 从"异步小工"变成"持续协作者"

如果 Jitro 兑现承诺，Jules 在 AI 编程工具里的定位会从"批量任务执行器"升级为"长期项目合作伙伴"。值得关注。
