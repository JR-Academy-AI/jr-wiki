n8n 是一个让你用可视化节点连接各种 App 和 API 的工作流自动化平台——从定时抓数据、发邮件通知，到构建有记忆的 AI Agent，全部在一个画布里搞定，不需要为每个集成单独写代码。

![n8n 工作流画布](https://n8n.io/images/n8n-screenshot.png)

和 Zapier 或 Make 这类纯 SaaS 工具不同，n8n 可以**完全自托管**：数据不出你的服务器，执行次数无上限，从零到跑起来只需要一条 Docker 命令。

## 2026年：n8n 2.0 带来了什么

2025年底到2026年初，n8n 发布了 2.0 大版本，对稳定性、AI 能力和安全性做了全面升级。几个值得关注的更新：

### Task Runners：Code 节点执行隔离

Code 节点现在运行在独立的 Task Runner 进程里，不再和 n8n 主进程共享内存：

- 一个写死循环的代码，不会把整个 n8n 实例搞崩
- 内存泄漏被隔离在 Task Runner，不影响其他工作流
- 多个 Code 节点可以并行执行，吞吐量提升明显

对生产环境来说，这个改动让 n8n 的稳定性上了一个台阶。

### Human-in-the-Loop：Send and Wait

这是 2026年 AI 工作流里最实用的新特性。AI Agent 在执行关键步骤时，可以暂停并向人类发消息等待确认，然后根据回复继续执行：

```
AI Agent 分析退款申请
    ↓
[Send and Wait] "这笔 $5,000 的退款申请已核实，确认要处理吗？"
    ↓（管理员在 Chat 界面点击 ✅ 确认 或 ❌ 拒绝）
根据回复：执行退款 or 拒绝并通知用户
```

对于不可逆操作——退款、删库、大量群发邮件——这个模式可以在 AI 出错时及时拦截，避免损失。

### 版本对比 UI

打开版本历史，选择任意两个版本，画布会并排展示差异，修改过的节点会高亮标记，旁边还有修改数量角标。再也不用靠「肉眼找不同」来 review 工作流改动了。

### 外部密钥管理（1Password）

生产环境的 API Key 现在可以存在 1Password Connect Server，n8n 在执行时实时拉取，不存入本地数据库。即使数据库泄漏，密钥也不受影响。还支持同时配置多个密钥提供商。

### 性能提升

大型实例（数百个工作流）的列表页加载速度提升了 30%–80%，历史执行记录的查询也快了不少。

## n8n vs Zapier vs Make：怎么选

| 维度 | n8n | Zapier | Make |
|------|-----|--------|------|
| **自托管** | ✅ | ❌ | ❌ |
| **自托管成本** | 仅 VPS 费用（$5–20/月） | N/A | N/A |
| **AI Agent** | 原生 LangChain，70+ AI 节点 | Zapier Agents（受限） | Maia（受限） |
| **代码能力** | 完整 JS + Python，支持 npm | 受限 | 受限 |
| **集成数量** | 1,000+ 原生 + 无限 HTTP | 8,000+ | 1,500+ |
| **MCP 协议** | ✅ 官方支持 | ❌ | ❌ |
| **Human-in-Loop** | ✅ Send and Wait | ❌ | 有限支持 |
| **价格（10k 执行/月）** | 自托管：免费；Cloud：€60/月 | $49/月 | $16/月 |
| **学习曲线** | 中（需要 JSON/API 基础） | 低 | 中 |

**选 n8n 的理由**：有一点技术背景、需要 AI Agent 深度集成、数据合规要求高（需自托管）、业务量大需要控制成本。

**选 Zapier 的理由**：完全非技术背景的团队、需要连接冷门 SaaS 工具（Zapier 8,000+ 集成覆盖最广）、不想折腾服务器。
