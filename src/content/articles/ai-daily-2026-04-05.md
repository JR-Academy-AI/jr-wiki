## 1. Google Gemma 4：Apache 2.0 开源，31B 拿下全球第三

**一句话概括**：Google DeepMind 在 4 月 2 日发布 Gemma 4 系列，四个尺寸（2B/4B/26B MoE/31B Dense），全部 Apache 2.0 开源，31B 在 Arena AI 文本榜排名全球开源第三。

**为什么重要**：Gemma 4 把开源模型的能力上限又推高了一档。31B 在 AIME 2026 数学竞赛基准上达到 89.2%，GPQA Diamond 科学推理 84.3%，LiveCodeBench v6 编程 80.0%——这些数字放在一年前都是闭源旗舰才能摸到的水准。

更关键的是 Apache 2.0 授权。对比 Meta Llama 4 的社区许可（有商业限制），Gemma 4 可以直接拿去做商业产品，不用担心被 Meta 的律师函绊倒。26B MoE 版本激活参数少，推理成本低，很可能成为一批创业公司的首选底座。

**对开发者的影响**：如果你在用 Llama 或 Qwen 做本地部署或微调，Gemma 4 现在值得认真测一遍。31B 在 Codeforces ELO 上从 110 飙到 2150，编程能力是真实的，不只是 PR。Google 同步在 Google AI Studio 和 Kaggle 上提供免费访问，上手成本为零。

原文链接：[Gemma 4: Byte for byte, the most capable open models](https://blog.google/innovation-and-ai/technology/developers-tools/gemma-4/)

## 2. 微软 MAI 三模型：Mustafa Suleyman 团队正式出手

**一句话概括**：微软 MAI Superintelligence 团队（CEO Mustafa Suleiman 主导）发布 MAI-Transcribe-1、MAI-Voice-1、MAI-Image-2，三个模态一次到位，通过 Azure Foundry 和 MAI Playground 提供服务。

**为什么重要**：这是微软第一次用"自研基础模型"而不是"调用 OpenAI API"的方式对外亮剑。MAI-Transcribe-1 在 25 种语言上的转录速度是 Azure Fast 的 2.5 倍；MAI-Voice-1 可以 1 秒生成 60 秒音频并支持自定义音色；MAI-Image-2 补完多模态矩阵。

微软用 400 亿美元买了 OpenAI 的股份，同时又在养一支和 OpenAI 直接竞争的团队——这不是矛盾，这是大公司的标准操作：永远不把鸡蛋放一个篮子。对 OpenAI 来说，客户如果既能用 ChatGPT 又能用 Azure MAI，议价能力就会被稀释。

**对开发者的影响**：如果你的产品需要语音转录或 TTS，MAI-Transcribe-1 和 MAI-Voice-1 现在是值得评估的选项，尤其是已经在 Azure 生态里的团队——切换成本几乎为零。

原文链接：[Microsoft launches 3 new AI models in direct shot at OpenAI and Google](https://venturebeat.com/technology/microsoft-launches-3-new-ai-models-in-direct-shot-at-openai-and-google)

## 3. Anthropic 融 300 亿，估值 3800 亿，Claude Code 年化 25 亿美元

**一句话概括**：Anthropic 在 2 月完成 Series G，融资 300 亿美元，估值 3800 亿美元（历史第二大风投交易），整体年化收入达 140 亿美元，Claude Code 单产品年化超 25 亿美元。

**为什么重要**：这些数字说明一件事：企业 AI 和 AI 编程助手已经是真实的大生意，不是 PPT。Anthropic 企业客户数从两年前不到 1000 家增长到 30 万以上，超 10 万美元年化账户一年涨了 7 倍。

更值得关注的是 Claude Code 的数据——AI 编程助手的天花板比大多数人想象的高。GitHub Copilot、Cursor 加上 Claude Code，这个赛道正在快速成熟。

**对开发者的影响**：Claude Code 的成功反映了开发者愿意为"真正有用的 AI 编程工具"付费。如果你还没用上 AI 辅助编程，竞争对手可能已经用上了。JR Academy 也有专门的 [Claude Code 工作流教程](/content/articles/claude-code-workflow) 供参考。

## 4. DeepSeek V4：本月发布，1 万亿参数 + 百万 Token 上下文

**一句话概括**：DeepSeek V4 预计 2026 年 4 月正式发布，采用万亿参数 MoE 架构（每次激活约 37B 参数），原生多模态，支持 100 万 Token 上下文，Apache 2.0 开源。

**为什么重要**：DeepSeek 上一个版本就已经在编程和数学上接近甚至超过部分闭源旗舰，V4 如果兑现宣传规格（SWE-bench 80%+、HumanEval 90%），将是开源阵营最强的编程模型之一。百万 Token 上下文意味着可以直接把整个代码库塞进去分析。

DeepSeek 的 MoE 路线非常聪明：1 万亿总参数但每次只激活 37B，推理成本和 37B 模型相当，但整体知识容量远超普通 37B。配合 Apache 2.0，这个组合对企业私有部署的吸引力很强。

**对开发者的影响**：关注 DeepSeek V4 的发布。如果基准数据属实，它会是本地部署的最强编程助手候选。中国 AI 大模型周调用量已经连续三周超过美国（7.359 万亿 Token vs 美国），DeepSeek 在这个趋势里扮演重要角色。

原文链接：[DeepSeek V4 And Tencent's New Hunyuan Model To Launch In April](https://dataconomy.com/2026/03/16/deepseek-v4-and-tencents-new-hunyuan-model-to-launch-in-april/)

## 5. Agentic AI Foundation 成立：MCP 成为行业标准的第一步

**一句话概括**：OpenAI、Anthropic、Block 联合在 Linux Foundation 下成立 Agentic AI Foundation；Anthropic 捐出 MCP 协议，OpenAI 捐出 Agents.md 规范，Block 捐出 Goose 框架，Google、微软、AWS 均已加入。

**为什么重要**：AI 智能体生态的碎片化问题终于被认真对待了。MCP（Model Context Protocol）已经被大量工具采用，现在进入 Linux Foundation 意味着它从 Anthropic 的私有协议变成了行业开放标准，和 HTTP、OAuth 在同一层级上被维护。

几个互相竞争的公司同时把自家的核心协议捐出来，这种事在 AI 行业很罕见。背后逻辑是：标准化对所有人都有好处，因为它扩大了整个市场，竞争可以在应用层进行，而不是在协议层内耗。

**对开发者的影响**：MCP 现在更值得认真学了——它不会消失，也不会被某家公司私有化。如果你在构建 AI 智能体或需要 LLM 连接外部工具，MCP 是当前最稳的选择。JR Academy 正在准备 MCP 实战课程，敬请关注。
