## 一、认证动态

### 1. DP-100 六月一日退场：倒计时 26 天，AI-300 本月正式上线接棒

**一句话**：传统数据科学认证走进历史，MLOps 工程师认证本月接棒，还没考 DP-100 的人直接备考 AI-300 更值。

微软 **DP-100（Designing and Implementing a Data Science Solution on Azure）** 确认于 **2026 年 6 月 1 日 17:00（美国中部时间）** 正式退场。今天是 5 月 6 日，6 月 1 日是 26 天后，这是 2026 年上半年所有微软认证退场日期里最近的一个。

替代方是 **AI-300（MLOps Engineer Associate）**，Beta 阶段于 2026 年 3 月发布，正式版（Live）预计**五月上线**，报名入口在 Microsoft Learn 认证页面。

**考点变了什么：**

DP-100 的核心是传统数据科学工作流：探索性数据分析、用 Azure Machine Learning Studio 训练和评估模型、部署推断端点。这些技能本身没有消失，但微软认为"会训练模型"这件事已经不够，AI-300 把考点重心大幅转移到模型生命周期的**工程化管理**：

- **CI/CD pipeline 自动化**：用 Azure DevOps 或 GitHub Actions 管理模型训练和部署流程，考实操配置，不只是概念
- **基础设施即代码（IaC）**：用 Bicep 或 Terraform 管理 Azure ML workspace 和相关资源
- **模型监控与漂移检测**：用 Azure Monitor 和 Application Insights 追踪 model drift，设置告警阈值
- **成本控制与资源治理**：Compute 集群配额管理、低优先级 VM 策略、实验成本归因

简单说：DP-100 考的是"我能训练一个模型"，AI-300 考的是"我能把 10 个模型的生产环境跑起来不出事"。

**行动建议：**

- 如果你已经在备考 DP-100 且进度 80% 以上：6 月 1 日前速战速决，否则切换到 AI-300 备考材料。
- 如果你才刚开始：直接备考 AI-300，DP-100 资料此刻已是过期投资。
- 已经拿到 DP-100 证书的：续证评估会改为针对 AI-300 考点，续证窗口期内在 Microsoft Learn 完成免费评估即可，不用重考。

### 2. AI-102 六月三十日退场：AI-103 把 Azure AI 工程师岗位重新定义为 Agent 开发者

**一句话**：调用 Azure Cognitive Services API 已经不够了，新考试要求你能构建和部署多步骤 AI Agent 应用。

**AI-102（Designing and Implementing a Microsoft Azure AI Solution）** 于 **2026 年 6 月 30 日** 退场，Beta 替代考试 **AI-103（Azure AI App and Agent Developer Associate）** 已于 **2026 年 4 月 21 日**开放报名，正式版（Live）预计 6 月上线。

AI-102 的定位是 Azure AI 服务的集成工程师：Language Understanding、Form Recognizer、Computer Vision、Azure OpenAI Service 怎么接进系统里。考点相对独立，各服务之间关联不强。

AI-103 的出发点完全不同，它假设你已经懂得调用各服务，重点考**怎么用这些能力构建端到端的 AI Agent 应用**：

- **Azure AI Foundry 工作流**：从模型选型、微调到 Agent 编排，全在 Foundry 平台上完成
- **Prompt Flow 和 Semantic Kernel**：构建多步骤推理链和工具调用（Tool Use）逻辑
- **RAG 架构实现**：Azure AI Search + Azure OpenAI 的完整检索增强生成 pipeline
- **Agent 安全与评估**：内容过滤（Content Safety）、Groundedness 检测、Responsible AI 集成

这个转变反映了市场现实：Copilot Studio 插件、Azure AI Foundry 上的 Agent 部署、以 Semantic Kernel 为中心的 AI 应用架构，是 2026 年澳洲企业客户最密集采购的几个方向，AI-103 的考点直接对标这些场景。

**行动建议：**

6 月 30 日前拿到 AI-102 的，证书有效期 1 年。但如果 AI-102 备考进度不足 60%，切换到 AI-103 更合适——同样的学习时间，拿到的证书方向更新，面试时信号更强。AI-103 Beta 阶段约 $33（正式价 $165 的 20%），Beta 名额有限，先到先得。

### 3. AI-900 六月三十日退场：AI-901 加入 Python 和 Microsoft Foundry，入门门槛提高

**一句话**：Azure AI 基础级认证不再只考概念，新版要求你真的写过 Python 代码和用过 Foundry 平台。

**AI-900（Microsoft Azure AI Fundamentals）** 确认与 AI-102 同日退场——**2026 年 6 月 30 日**。Beta 替代考试 **AI-901** 同样于 **2026 年 4 月 21 日**开放。

两代考试的核心差异在定位：

- **AI-900 的逻辑**：面向完全没有技术背景的人，考 Azure AI 服务的概念知识——什么是机器学习、什么是 NLP、Language Service / Vision Service / Bot Service 的适用场景。无需编程背景，选择题为主。

- **AI-901 的逻辑**：仍是基础级（Fundamentals），但微软把基础线拉高了。新版要求**熟悉 Python 编程语法**，因为考点从"认识各 AI 服务"变成了"在 Azure AI Foundry 上用代码实现 AI 解决方案"，包括部署生成式 AI 模型、用 Prompt Flow 构建简单 Agent。AI-901 特别点明，考生需要"熟悉 Python 编程技巧"。

对于只想刷入门证书的人，这个变化意味着准备时间要增加。Microsoft Learn 上已经有 AI-901 的官方免费学习路径（详见下方免费课程第三条）。

已持有 AI-900 的：微软基础级认证**永久有效**，AI-900 不会从你的 Microsoft Learn 成绩单上消失，无需任何操作。

## 二、免费 / 限免课程

### 1. Oracle + DeepLearning.AI：Agent Memory — Building Memory-Aware Agents（免费）

**适合**：已经在构建 AI Agent 的工程师，想让 Agent 跨对话记住用户偏好和任务历史。
**链接**：[deeplearning.ai/short-courses/](https://www.deeplearning.ai/short-courses/) 或 [Oracle Developer Blog](https://blogs.oracle.com/developers/oracle-and-deeplearning-ai-launch-new-agent-memory-course-for-ai-developers)

Oracle 和 DeepLearning.AI 联合发布这门免费短课，教如何用 Oracle AI Database 为 AI Agent 构建记忆系统。课程聚焦于一个实际痛点：大多数基于 LLM 的 Agent 每次对话都从零开始，无法记住用户上一次说了什么、任务做到哪一步、有什么偏好。

课程涵盖的记忆类型：
- **短期记忆（Episodic）**：当前对话上下文窗口管理
- **长期记忆（Semantic）**：跨对话持久化存储，用向量检索召回历史
- **过程记忆（Procedural）**：Agent 学习任务执行步骤的规律

技术实现用 Oracle AI Vector Database 存储记忆条目，结合 Oracle's AI Infrastructure。课程是 DeepLearning.AI 一贯的短课形式，1-2 小时完成，全程免费，无需信用卡。适合已经在用 LangChain、AutoGen 或 CrewAI 搭 Agent 的人，直接学可部署的记忆架构。

### 2. Google Skills：Develop Agents with ADK 免费路径（含 Skill Badge）

**适合**：想用 Google 官方框架构建和部署 AI Agent 的开发者，从零到上手 Agent Development Kit。
**链接**：[skills.google.com/paths/3545](https://www.skills.google/paths/3545)（Develop Agents with Agent Development Kit）

Google 在 Google Skills 平台上线了完整的 **ADK（Agent Development Kit）** 学习路径，涵盖三个层次：

1. **Get Started with ADK**：ADK 基础概念，跑通第一个 Agent，无需深厚 Python 背景
2. **Engineer AI Agents with ADK**：多模态 Agent 构建，Tool Use 配置，Agent 之间协作编排
3. **Build Intelligent Agents with ADK**：复杂多 Agent 系统设计，生产部署

学完路径可获得 **Google Cloud Skill Badge**，这是 Google 官方的技能徽章，显示在 LinkedIn 和简历上有背书价值。整条路径免费，Labs 部分需要 Google Skills Credits，大学生可凭大学邮箱申请 200 Credits 免费用，其他人可选每月 35 Credits 的 GEAR Program（需单独报名）。

Google ADK 是 2026 年 4 月开源的框架，定位是和 LangGraph、CrewAI 竞争的 Google 官方 Agent 编排方案，支持 Gemini 模型和 Google Cloud 生态集成。如果你的工作场景涉及 Google Cloud，直接用官方框架比三方工具更稳定。

> 来源：[skills.google.com — Develop Agents with ADK](https://www.skills.google/paths/3545) · [Google Free Agent Development Kit Course 2026](https://www.coursejoin.com/free-online-courses/google-free-agent-development-kit-course/)

### 3. DeepLearning.AI：Building Live Voice Agents with Google's ADK（免费短课）

**适合**：想做语音交互 AI Agent 的开发者，特别是需要低延迟实时语音响应的场景。
**链接**：[deeplearning.ai/short-courses/building-live-voice-agents-with-googles-adk/](https://www.deeplearning.ai/short-courses/building-live-voice-agents-with-googles-adk/)

这门课由两位 Google Machine Learning Engineer **Lavi Nigam** 和 **Sita Lakshmi Sangameswaran** 主讲，专注于实时语音 Agent 的构建和部署。与上一条 Google Skills 路径偏系统设计不同，这门课是端到端动手实操：从语音信号捕获、转录，到 LLM 推理，再到语音合成和响应输出，全链路用 ADK 串起来。

课程重点：
- 用 ADK 的 Live API 处理音频流，实现低于 500ms 的响应延迟
- 多轮语音对话中的上下文管理
- 在 Google Cloud 上部署语音 Agent 并接入真实电话或 WebRTC

这门课面向的是有 Python 基础、想做语音产品的开发者，不是纯理论课。DeepLearning.AI 短课全部免费，视频 + 代码均可访问，无需订阅。

> 来源：[deeplearning.ai — Building Live Voice Agents with Google's ADK](https://www.deeplearning.ai/short-courses/building-live-voice-agents-with-googles-adk/)

**1. DP-100 备考者：本周决策，别拖到第 25 天**

AI-900 6 月 30 日退场，AI-901 已经开放 Beta，正式版 6 月到位。入门学员直接备考 AI-901，官方免费学习路径在 [learn.microsoft.com/en-us/credentials/certifications/azure-ai-fundamentals/](https://learn.microsoft.com/en-us/credentials/certifications/azure-ai-fundamentals/) 可以找到。

打开 [Microsoft Learn 认证页面](https://learn.microsoft.com/en-us/credentials/certifications/)，搜索你持有或正在备考的所有认证，确认退场状态。2026 年 4 月到 9 月集中退场 7 张，别让手头的备考资料变成备用纸。
